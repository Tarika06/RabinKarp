Rabin-Karp Algorithm

What is it?

  • A string searching algorithm that uses hashing to find pattern occurrences in text efficiently.
  • Uses a rolling hash ‚Äî updating the hash in O(1) per step instead of recomputing from scratch.
  • When hashes match, it performs a full character-by-character verification to confirm (avoiding false positives from hash collisions).

Rolling Hash Formula

hash(s[0..m-1]) = Œ£ s[i] √ó BASE^(m-1-i) mod PRIME

Rolling update:
newHash = (BASE √ó (oldHash ‚àí s[i] √ó BASE^(m‚àí1))
         + s[i + m]) mod PRIME

BASE = 256  |  PRIME = 101

Time Complexity

Case Complexity
Best Case O(n + m)
Average Case O(n + m)
Worst Case O(n √ó m)

Where n = text length,  m = pattern length.

‚ö† Hash Collisions: Two different strings can produce the same hash value. That's why Rabin-Karp always verifies a hash match with a full string comparison. A hash match without a string match is called a false positive (also called a spurious hit).
